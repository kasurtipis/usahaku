Documentation and interactive API exploration for UsahaKu Blog Platform

Welcome to UsahaKu API

UsahaKu API provides a complete interface for managing blog content, categories, and comments in our multi-tenant blog platform. This API follows RESTful principles and returns JSON responses.

Base URL

All API requests should be made to the following base URL:

https://yourdomain.com/api/v1

Authentication

Some endpoints require authentication using a Bearer token. Include your API token in the Authorization header:

Authorization: Bearer YOUR_API_TOKEN

Endpoints marked with Auth Required require authentication.

Blog Categories API

Manage blog categories in your application.

GET /blog-categories Auth Required

Retrieve a paginated list of blog categories.

POST /blog-categories Auth Required

Create a new blog category.

Required fields: username, subdomain, title, slug

GET /blog-categories/{id}

Retrieve a specific blog category by ID.

PUT /blog-categories/{id} Auth Required

Update an existing blog category.

DELETE /blog-categories/{id} Auth Required

Delete an existing blog category.

Blogs API

Manage blog posts in your application.

GET /blogs

Retrieve a paginated list of blogs.

POST /blogs Auth Required

Create a new blog post.

Required fields: slug, subdomain, username, title, blog_category_id, author, content

GET /blogs/{id}

Retrieve a specific blog by ID.

PUT /blogs/{id} Auth Required

Update an existing blog.

DELETE /blogs/{id} Auth Required

Delete an existing blog.

GET /blogs/category/{category}

Retrieve blogs filtered by category ID.

GET /blogs/author/{author}

Retrieve blogs filtered by author.

Blog Comments API

Manage comments on blog posts.

GET /blog-comments

Retrieve a paginated list of blog comments.

POST /blog-comments

Create a new blog comment.

Required fields: blog_id, username, subdomain, content

GET /blog-comments/{id}

Retrieve a specific blog comment by ID.

PUT /blog-comments/{id} Auth Required

Update an existing blog comment.

DELETE /blog-comments/{id} Auth Required

Delete an existing blog comment.

Error Handling

Our API returns standard HTTP status codes. Common error codes include:

  • 200 OK - Request successful
  • 201 Created - Resource successfully created
  • 204 No Content - Request successful, no content to return
  • 400 Bad Request - Request was malformed
  • 401 Unauthorized - Authentication required
  • 403 Forbidden - Access to resource denied
  • 404 Not Found - Resource doesn't exist
  • 422 Unprocessable Entity - Validation error
  • 500 Internal Server Error - Server error
